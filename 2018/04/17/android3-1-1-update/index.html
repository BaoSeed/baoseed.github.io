报错：

/Users/Ivan/Desktop/zixun/android/app/build/intermediates/res/merged/release/drawable-xhdpi/app_view_imgs_financial_name.png: error: uncompiled PNG file passed as argument. Must be compiled first into .flat file..

github上的issue:

This is what I discovered:

Only happens for release builds.
This issues seems to stem from duplicate assets in the drawable folder: fails with the error “Duplicate file”
To make it work, before running gradlew assembleRelease delete all images from

‘android/app/src/main/res/drawable-mdpi’
‘android/app/src/main/res/drawable-hdpi’
‘android/app/src/main/res/drawable-xhdpi’
‘android/app/src/main/res/drawable-xxhdpi’
‘android/app/src/main/res/drawable-xxxhdpi’
Or, generate signed APKs using android studio. That works as well for some reason.

参考 xxx uncompiled PNG file passed as argument. Must be compiled first into .flat file.
